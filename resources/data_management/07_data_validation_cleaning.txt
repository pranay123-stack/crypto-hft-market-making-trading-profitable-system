DATA VALIDATION AND CLEANING FOR HFT SYSTEMS
================================================================================

VALIDATION FRAMEWORK
-------------------
Multi-layer validation: Ingestion, Business Logic, Cross-Reference, Statistical

C++ Validation Engine with rules:
- PriceRangeValidator: Range and change validation
- SpreadValidator: Bid/ask spread checks
- VolumeValidator: Volume spike detection
- SequenceValidator: Gap detection
- TimestampValidator: Latency and future timestamp checks

CLEANING STRATEGIES
------------------
Outlier Detection:
- Z-score method (threshold: 3 sigma)
- IQR method (1.5 * IQR from quartiles)
- Rolling window statistics

Gap Filling:
- Linear interpolation
- Forward/backward fill
- Cubic spline
- Mean imputation

Duplicate Detection:
- Fingerprint-based deduplication
- Time-window sliding
- Hash-based detection

CROSS-FEED VALIDATION
--------------------
Multi-source comparison with consensus pricing
Deviation threshold: 1% from median
Recent data weighting (5-second window)

QUALITY MONITORING
-----------------
Metrics collection:
- Validity rate
- Duplicate rate
- Outlier rate
- Gap fill rate
- Validation latency (P50, P95, P99)

SQL VALIDATION QUERIES
---------------------

position reconciliation, gap filling queries

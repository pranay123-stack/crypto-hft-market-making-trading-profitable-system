1. EXECUTIVE SUMMARY
================================================================================

Server selection is foundational to HFT infrastructure performance. The right
platform balances CPU performance, memory bandwidth, PCIe connectivity, and
power efficiency while meeting strict latency requirements.

Critical Selection Factors for HFT:
- CPU: Latest generation Intel Xeon or AMD EPYC processors
- Memory: DDR5-4800 or faster, ECC required, minimum 256GB
- PCIe: Gen 4.0/5.0 support with sufficient lanes for NICs and FPGAs
- Storage: NVMe for low-latency data access
- Power: Redundant PSUs with 80+ Titanium efficiency
- Management: Out-of-band management (iDRAC, iLO, IPMI)
- Form Factor: 1U or 2U rack-mount for density

Market Landscape 2024:
Dell and HPE dominate the enterprise market with comprehensive support,
while Supermicro offers competitive pricing and flexibility. Intel Xeon
Scalable (Sapphire Rapids) and AMD EPYC (Genoa) processors deliver
exceptional performance for latency-sensitive workloads.

Typical HFT Deployment:
- Trading servers: 2-4 high-performance 1U/2U servers per strategy
- Market data servers: 4-8 servers with high-bandwidth NICs
- Risk/analytics: 2-4 servers with large memory configurations
- Development: 2-4 servers matching production specs

Budget Ranges:
- Entry-level (development): $8,000 - $10,000 per server
- Mid-range (production): $12,000 - $18,000 per server
- High-end (ultra-low latency): $20,000 - $35,000 per server

INSTALLATION AND DEPLOYMENT
----------------------------
Initial Setup:
1. Rack installation with provided rail kit (2-person job)
2. Cable management arms recommended for dense deployments
3. Power: Dual feed from separate PDUs for redundancy
4. Network: Minimum 4 connections (2 data, 2 management)

BIOS Configuration for HFT:
- Boot Mode: UEFI
- Processor Settings:
  - Logical Processor: Enabled (or disabled for dedicated cores)
  - Virtualization Technology: Disabled (unless using VMs)
  - Hardware Prefetcher: Enabled
  - Adjacent Cache Line Prefetch: Enabled
  - C-States: Disabled (force maximum performance)
  - Turbo Boost: Enabled
  - Energy Performance: Maximum Performance
- Memory Settings:
  - Memory Operating Mode: Optimizer Mode
  - Node Interleaving: Disabled (NUMA awareness)
- PCI Settings:
  - SR-IOV: Enabled (if using VFs)
  - Memory Mapped I/O: Above 4GB
  - PCI ASPM L1: Disabled (no power saving)

iDRAC Configuration:
- Assign static IP on management network
- Enable SSH for scripted management
- Configure SNMP traps for monitoring
- Set up email alerts for hardware failures
- Enable virtual console for remote troubleshooting

Operating System Installation:
Recommended: Ubuntu 22.04 LTS or Red Hat Enterprise Linux 8.x

1. Create RAID 1 for boot drives via PERC controller
2. Boot from USB/virtual media via iDRAC
3. Partition scheme:
   - /boot: 1GB
   - /: 100GB
   - /var: 50GB
   - /home: 50GB
   - /data: Remaining space on data NVMe drives
4. Install minimal server (no GUI)
5. Apply all updates: apt update && apt upgrade

Post-Install Optimization:
- Install Dell OpenManage tools: apt install srvadmin-all
- Update firmware: omconfig chassis bios
- Configure NUMA: numactl --hardware
- CPU frequency governor: cpupower frequency-set -g performance
- Disable transparent hugepages
- Configure kernel parameters (see section 10)

   - Verify rail lock mechanisms function

   - Slide server into rails (2 people required, 25-28 kg)
   - Secure with front mounting screws

   - Power: Connect to separate PDUs (A/B power)
   - Management: Connect to out-of-band management network
   - Data: Connect NICs per network design
   - Console: USB or serial for emergency access

BIOS CONFIGURATION FOR HFT
---------------------------
Dell PowerEdge R750 Example:

1. Boot into BIOS: F2 during POST

2. System Setup -> System BIOS:
   - Boot Mode: UEFI (not Legacy)
   - Boot Sequence: NVMe drive (OS)

3. Processor Settings:
   - Logical Processor: Disabled (for dedicated cores)
     OR Enabled (if using all logical cores)
   - Virtualization Technology: Disabled (no virtualization overhead)
   - Intel Turbo Boost: Enabled (maximum single-thread performance)
   - C-States: Disabled (prevent CPU sleep states)
   - C1E: Disabled
   - Execute Disable: Enabled (security)
   - Hardware Prefetcher: Enabled
   - Adjacent Cache Line Prefetch: Enabled
   - Energy Performance: Maximum Performance

4. Memory Settings:
   - Memory Operating Mode: Optimizer Mode (NUMA-aware)
   - Node Interleaving: Disabled (preserve NUMA topology)
   - Memory Patrol Scrub: Disabled (reduce background activity)

5. Integrated Devices:
   - SR-IOV Global Enable: Enabled (if using SR-IOV for NICs)
   - I/O Virtualization: Enabled (Intel VT-d/AMD-Vi)
   - USB Ports: Disabled (except management, reduce attack surface)

6. Serial Communication:
   - Serial Port Address: COM1 (for console access)

7. System Profile Settings:
   - System Profile: Performance (not Energy Efficient)

8. PCI Settings:
   - Memory Mapped I/O above 4GB: Enabled (large NIC BARs)
   - PCI ASPM L1 Link Power Management: Disabled
   - SR-IOV Support: Enabled

9. Save and Exit: F10

OPERATING SYSTEM INSTALLATION
------------------------------
Recommended OS for HFT: Ubuntu 22.04 LTS or Red Hat Enterprise Linux 8.x

Ubuntu 22.04 LTS Installation:

1. Create Installation Media:
   - Download Ubuntu Server 22.04 LTS ISO
   - Mount via iDRAC/iLO virtual media OR USB drive

2. Boot from Installation Media:
   - Boot order: USB/Virtual media first

3. Installation Steps:
   - Language: English
   - Keyboard: US
   - Network: Configure all interfaces with static IPs
   - Storage: Manual partitioning (recommended):

     Partition Scheme (NVMe boot drives in RAID 1):
     /boot/efi:  1GB   (EFI System Partition)
     /boot:      2GB   (ext4)
     /:          100GB (ext4 or XFS)
     /var:       50GB  (ext4 or XFS)
     /home:      20GB  (ext4)
     swap:       64GB  (equal to RAM if < 64GB, else 64GB)
     /data:      remaining on data NVMe drives (XFS recommended)

   - Profile: Minimal Ubuntu Server (no desktop)
   - SSH: Install OpenSSH server
   - Packages: None (install later)

4. First Boot Configuration:
   sudo apt update && sudo apt upgrade -y
   sudo apt install build-essential linux-headers-$(uname -r)
   sudo apt install net-tools ethtool sysstat htop iotop

5. Kernel Parameters:
   Edit /etc/default/grub:

   GRUB_CMDLINE_LINUX="intel_iommu=on iommu=pt default_hugepagesz=1G \
   hugepagesz=1G hugepages=64 isolcpus=8-63 nohz_full=8-63 rcu_nocbs=8-63 \
   processor.max_cstate=0 intel_idle.max_cstate=0 idle=poll \
   tsc=reliable clocksource=tsc"

   Update GRUB:
   sudo update-grub
   sudo reboot

6. System Tuning:
   Create /etc/sysctl.d/99-hft-tuning.conf:

   # Network stack tuning
   net.core.rmem_max = 134217728
   net.core.wmem_max = 134217728
   net.core.netdev_max_backlog = 250000
   net.ipv4.tcp_rmem = 4096 87380 134217728
   net.ipv4.tcp_wmem = 4096 65536 134217728
   net.ipv4.tcp_timestamps = 0
   net.ipv4.tcp_sack = 0

   # Connection tracking
   net.netfilter.nf_conntrack_max = 2000000

   # Disable swap
   vm.swappiness = 0

   # Shared memory
   kernel.shmmax = 68719476736
   kernel.shmall = 4294967296

   Apply:
   sudo sysctl -p /etc/sysctl.d/99-hft-tuning.conf

7. Disable Unnecessary Services:
   sudo systemctl disable bluetooth.service
   sudo systemctl disable cups.service
   sudo systemctl disable avahi-daemon.service
   sudo systemctl mask systemd-resolved.service

8. CPU Frequency Scaling:
   sudo apt install linux-tools-common linux-tools-$(uname -r)

   Set performance governor:
   for cpu in /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor; do
       echo performance | sudo tee $cpu
   done

   Make persistent:
   sudo apt install cpufrequtils
   echo 'GOVERNOR="performance"' | sudo tee /etc/default/cpufrequtils
   sudo systemctl restart cpufrequtils

HARDWARE VALIDATION
--------------------
1. CPU Testing:
   - Install stress-ng: sudo apt install stress-ng
   - CPU stress test: stress-ng --cpu $(nproc) --timeout 3600s
   - Monitor temperatures: watch -n 1 sensors
   - Verify no thermal throttling

2. Memory Testing:
   - Install memtester: sudo apt install memtester
   - Test memory: sudo memtester 100G 1
   - Run overnight for production servers

3. Storage Testing:
   - Install fio: sudo apt install fio
   - NVMe read test:
     fio --name=seqread --rw=read --bs=1M --size=10G --numjobs=1 \
     --runtime=60 --time_based --filename=/data/testfile

   - NVMe write test:
     fio --name=seqwrite --rw=write --bs=1M --size=10G --numjobs=1 \
     --runtime=60 --time_based --filename=/data/testfile

   - Random IOPS:
     fio --name=randrw --rw=randrw --bs=4k --size=10G --numjobs=16 \
     --runtime=60 --time_based --filename=/data/testfile

4. Network Testing:
   - Install tools: sudo apt install iperf3 netperf
   - Throughput test: iperf3 -c <remote_host> -t 60 -P 4
   - Latency test: sockperf ping-pong -i <remote_host> -p 12000
   - Verify line rate and sub-microsecond latency

   - Faster delivery (some stock on hand)
   - NET30 payment terms available
   - Relationship pricing for repeat customers
   - One-stop shopping for multiple brands

   Disadvantages:
   - 5-10% markup vs. direct
   - Limited custom configurations
   - Support goes through reseller first

3. Grey Market / Refurbished:
   Advantages:
   - 30-50% cost savings
   - Immediate availability

   Disadvantages:
   - Limited or no warranty
   - Unknown component history
   - Not recommended for production HFT systems

RFP (REQUEST FOR PROPOSAL) STRATEGY
------------------------------------
For deployments of 10+ servers:

1. Prepare RFP Document:
   - Exact specifications required
   - Delivery timeline
   - Support requirements
   - Payment terms
   - Total 3-year TCO

2. Send to Multiple Vendors:
   - Dell
   - HPE
   - Supermicro (via integrator)
   - Lenovo

3. Evaluation Criteria:
   - Price (40%)
   - Lead time (20%)
   - Support quality (20%)
   - Vendor relationship (10%)
   - Financing options (10%)

4. Negotiation Points:
   - Volume discounts
   - Extended payment terms (NET60/NET90)
   - Free extended warranty
   - Free advanced replacement
   - On-site technical support for deployment

FINANCING OPTIONS
------------------
1. Dell Financial Services (DFS):
   - 3-year leases available
   - Fair Market Value (FMV) lease: Lower monthly payments
   - $1 Buyout: Fixed end-of-lease cost
   - Operating lease: Off-balance sheet treatment

   Example: $150,000 server purchase
   - 36-month FMV lease: $4,800/month
   - Total cost: $172,800 (15% premium)

2. HP Financial Services:
   - Similar structure to DFS
   - Technology refresh programs
   - Flexible end-of-term options

3. Third-Party Leasing:
   - GreatAmerica Financial Services
   - CIT Group
   - Typically 1-2% higher rates than OEM

SPARE PARTS STRATEGY
--------------------
Recommended spares for 10-server deployment:

Critical Spares (keep on-site):
- 2x Power supplies (most common failure): $500 each
- 4x DIMMs (memory failures): $300 each
- 2x NVMe drives (boot drives): $200 each
- 1x Network card (backup for trading server): $2,500

Total Spares Investment: $5,100

Non-Critical (vendor advanced replacement):
- Motherboard
- CPUs
- Fans
- RAID controllers

Justification:
- 4-hour vendor onsite may miss trading hours
- Critical failures need immediate resolution
- Spares pay for themselves after one avoided outage

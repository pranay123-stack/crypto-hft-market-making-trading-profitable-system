2.4 LICENSING TERMS & RESTRICTIONS
-----------------------------------

Named User Model:
- Each terminal assigned to specific individual
- Limited sharing (vacation coverage allowed)
- Non-transferable without notification
- Remote access: Requires 2-factor auth (built-in)

Concurrent User Model:
- Available for large enterprises (typically 100+ licenses)
- Priced at ~1.3x named user cost
- Example: 50 concurrent = equivalent to 65 named users
- Cost: 50 x $24K x 1.3 = $1,560,000/year

Data Usage Restrictions:
1. Display Use: View on terminal only
2. Non-Display Use: Automated trading (requires separate license)
3. Derived Data: Cannot create competing products
4. Redistribution: Prohibited without BDL agreement
5. Storage: Limited to internal operational needs
6. Latency: Standard terminal = 100-300ms; B-PIPE can be <10ms

API Usage Limits:
- Standard: 500 requests/hour per user
- High-volume: Custom limits (negotiated)
- Market data subscriptions: Per-symbol charges apply
- Historical data: Throttled unless BDL licensed

3.8 LICENSING TERMS
-------------------

Contract Terms:
- Minimum: 1 year (typically 3 years)
- Payment: Annual or quarterly
- Auto-renewal: Common (30-90 day notice required)
- Termination: Difficult mid-contract

Data Usage Rights:
1. Display: Desktop viewing
2. Non-Display: Automated systems (extra fee: +50-100%)
3. Derived Works: Restricted (requires separate agreement)
4. Redistribution: Prohibited
5. Storage: 30-90 days typical limit

Non-Display Fees:
- Algorithmic trading: +75% of display cost
- Systematic internalization: +100%
- Client reporting: Negotiable
- Example: $22K display → $38.5K total with non-display

Audit Rights:
- LSEG can audit usage annually
- Must maintain logs of data access
- Penalties for violations: 2-5x license fees
- Common violation: Exceeding user counts

9.2 NEGOTIATION STRATEGIES
---------------------------

Key Negotiation Points:

1. Volume Discounts:
- "We expect to scale from 10 to 30 users in 2 years"
- Request tiered pricing locked in advance
- Expected savings: 15-30%

2. Multi-Year Commits:
- Offer 3-year contract for 20-25% total discount
- Lock in pricing (annual 3-5% escalation max)
- Include growth provisions

3. Bundling:
- Combine multiple products from same vendor
- Example: Bloomberg Terminal + B-PIPE + BDL
- Expected savings: 10-20%

4. Payment Terms:
- Request NET 90 vs. NET 30
- Annual vs. quarterly (annual = 3-5% discount)
- Upfront payment (10-15% discount possible)

5. Pilot Programs:
- Free trial: 30-90 days
- Reduced pricing for first 6 months
- No commitment during pilot

6. Exit Clauses:
- Performance guarantees
- 30-60 day termination option (first year)
- Refund policy for non-performance

7. Competitive Pressure:
- "We're also evaluating [competitor]"
- Request price match
- Use competing quotes as leverage

Typical Savings:
- Initial quote: $500,000/year
- After negotiation: $350,000-$400,000/year
- Savings: 20-30%

9.3 CONTRACT TERMS TO NEGOTIATE
--------------------------------

Critical Clauses:

1. Auto-Renewal:
- Insist on 90-180 day notice period
- No automatic price increases >5%/year
- Option to reduce scope/users

2. Audit Rights:
- Limit vendor audit frequency (annual max)
- Reasonable notice (30+ days)
- Limit scope to relevant usage only

3. Data Ownership:
- Retain ownership of all uploaded data
- Right to export data upon termination
- No vendor usage of client data for other purposes

4. SLA & Credits:
┌────────────────────┬──────────┬──────────────┐
│ Uptime             │ Credit   │ Notes        │
├────────────────────┼──────────┼──────────────┤
│ 99.99% (promised)  │ 0%       │ Baseline     │
│ 99.9-99.99%        │ 10%      │ Minor breach │
│ 99.5-99.9%         │ 25%      │ Moderate     │
│ 99-99.5%           │ 50%      │ Serious      │
│ <99%               │ 100%     │ Major breach │
└────────────────────┴──────────┴──────────────┘

5. Limitation of Liability:
- Vendor typically caps at 12 months fees
- Negotiate for uncapped liability for:
  - Gross negligence
  - Data breaches
  - Willful misconduct
- Seek insurance coverage proof

6. Termination for Convenience:
- Right to terminate with 60-90 days notice
- Prorated refund for unused portion
- No termination fees

7. Price Protection:
- Cap annual increases at 3-5%
- Lock in volume discount tiers
- Most-favored-nation clause (match any better deals)

8. Implementation Support:
- Specify included hours (e.g., 100 hours)
- Additional hours capped at $150-$250/hour
- Dedicated support engineer during first 90 days

10.4 COST OPTIMIZATION STRATEGIES
----------------------------------

1. Consolidate Vendors:
- Reduce from 15 to 8-10 vendors
- Negotiate master service agreements
- Expected savings: 15-20%

2. Right-Size Licenses:
- Audit actual usage vs. purchased seats
- Convert named users to concurrent (where applicable)
- Expected savings: 10-15%

3. Negotiate Multi-Year Contracts:
- Commit 3 years for 20-25% discount
- Lock in pricing (max 5% annual escalation)
- Expected savings: 20-25% over 3 years

4. Use Open-Source Where Possible:
- Databases: PostgreSQL vs. Oracle
- Programming tools: Free IDEs
- Development frameworks: Open source
- Expected savings: $50,000-$200,000/year

5. Cloud vs. On-Premise TCO:
- Cloud: Higher monthly, zero CapEx
- On-premise: Higher upfront, lower long-term
- Hybrid approach often optimal
- Expected savings: 20-30% (hybrid vs. pure cloud)

6. Alternative Data Evaluation:
- Proof of alpha before committing
- Start with limited datasets
- Build internally where feasible
- Expected savings: 30-50% vs. buying all datasets

7. Internal Development:
- Build vs. buy analysis for each component
- Consider opportunity cost
- Typical breakeven: 2-4 years for major systems
